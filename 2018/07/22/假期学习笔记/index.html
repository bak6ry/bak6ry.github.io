xpath处理函数(extractvalue updatexml)

从mysql5.1.5开始提供的两个xml查询和修改的函数，其中extractvalue负责在xml文档中按照xpath语法查询节点=内容，updataxml则负责修改查询到的内容

基于主键重复 –http://drops.xmd5.com/static/drops/tips-14312.html

floor(rand(0)2) 报错条件是必须要有3条以上的查询数据，而且是一定会报错
被计算多次的意思就是，在使用的group by 的时候，floor(rand(0)2)会被执行一次，如果虚表不存在查询数据的话，在插入虚表之前，将会被再计算一次
rand(0)2报错原理 – 它的结果是固定的 011011…
首先建立了一个虚表
取第一条数据，执行floor(rand(0)2) 结果为0 查询虚表，发现键值为0不存在，再进行一次运算，结果为1，插入到表中
取第二条数据，执行，结果为1，查查询虚表，，纪录存在 直接count加一
取第三条数据，执行，结果为0 ，查询虚表，不存在键值0，再进行一次运算，结果为1，插入到虚表中作为虚表的主键，于之前的1冲突，这时候就会报错了
floor(rand()2) 在查询结果只有两条也可以报错，但是前提就是前面的查询不能让虚表中存在0和1两个键值
select count() from information_schema.tables group by floor(rand(0)*2);
ERROR: dduplicate entry ‘1’ for key ‘group_key’

基于数据类型的不一致

updataxml(1,xpath,1)函数的第二个参数要求为xpath格式，如果我们把它改为字符串，那么就会爆出xpath syntax error的错误，于是构造sql语句

select from user where id=1 updataxml(1,(concat(1,user())),1);
同样的函数还有extractvalue(1,xpath)
select from user where id=1 extractxml(1,(concat(1,user())));
